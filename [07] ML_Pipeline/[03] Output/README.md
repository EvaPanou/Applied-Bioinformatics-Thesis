# SLE ML Pipeline — Output Summary

This folder contains the main output files generated by the donor-stratified machine-learning pipeline developed to identify and evaluate a compact diagnostic gene signature for systemic lupus erythematosus (SLE).

## Pipeline Overview

The dataset contains repeated samples from the same donors across multiple timepoints. If samples from the same donor were split across training and testing subsets, the resulting performance estimates could be optimistically biased due to donor-level information leakage.

To reduce this risk, donor identity was preserved during splitting and validation. The held-out donor set was sealed after the initial split and was not used for feature selection, classifier tuning, benchmarking, or final panel construction. This approach is aligned with general recommendations for avoiding selection bias and overfitting in prediction-model development, especially in high-dimensional gene-expression settings [1–3].

The main pipeline stages are:

```text
Input DEG + metadata matrix
        ↓
Donor-aware 80/20 development/held-out split
        ↓
Development-set feature-selector/classifier benchmarking
        ↓
Final compact gene-panel derivation
        ↓
Sealed held-out evaluation
        ↓
Uncertainty analysis and SHAP-based interpretation
```

## Section 2 — Development-Set Algorithm Benchmarking

Only the development cohort was used during algorithm benchmarking. The aim of this stage was to compare different feature-selector/classifier combinations while preventing information leakage from the held-out cohort.

The development benchmark used an outer donor-aware 5-fold cross-validation scheme. Within each outer training split, a bootstrap Random Forest stability filter was applied before feature selection. This step prioritized genes that were repeatedly selected across donor-level resampling iterations, following the general rationale of stability-based selection strategies [4].

Within each outer fold:

1. The outer training donors were used for bootstrap RF stability filtering.
2. Each feature-selection method was applied to the stabilized training data.
3. Each classifier was tuned using inner donor-aware cross-validation.
4. The tuned model was refitted on the outer training donors.
5. The model was evaluated on the outer testing donors.
6. Outer-fold ROC AUC values were collected.

This nested structure reduces the risk of biased performance estimates caused by tuning or feature selection outside the validation loop [2,3].

## Section 3 — Final Gene-Panel Derivation

After selecting the best-performing feature-selector/classifier combination, the winning selector was rerun on the full development cohort to obtain a global ranking of candidate genes.

The final gene-panel derivation used:

- Boruta-based feature ranking;
- full-development bootstrap RF stability analysis;
- high-frequency gene retention;
- Pearson correlation-based redundancy pruning.

This stage was designed to derive a compact panel that preserved predictive signal while reducing redundancy among highly correlated genes. Correlation-based pruning is commonly used in high-dimensional modelling to avoid retaining multiple highly similar predictors.

## Methodological Safeguards

Several safeguards were used to reduce overfitting and biased performance estimation:

1. **Donor-aware splitting**  
   Samples from the same donor were kept together during development/held-out splitting and cross-validation.

2. **Sealed held-out evaluation**  
   The held-out cohort was not used during feature selection, model tuning, or final panel construction.

3. **Nested cross-validation during benchmarking**  
   Inner cross-validation was used for hyperparameter tuning, while outer cross-validation was used for performance estimation.

4. **Bootstrap RF stability filtering**  
   Repeated bootstrap resampling was used to prioritize stable candidate genes.

5. **Correlation-based redundancy pruning**  
   Pearson correlation filtering reduced redundancy in the final panel.

6. **Permutation testing**  
   A null AUC distribution was generated to assess whether observed model performance exceeded chance-level expectations.

7. **Transparent comparator model**  
   A logistic-regression comparator was trained and evaluated on the same final gene panel.

8. **SHAP interpretation**  
   SHAP analysis was used to summarize feature contributions for the final model.

## Notes and Limitations

This README reports the outputs generated by the current pipeline run. It does not provide a full biological interpretation of the selected genes or a full thesis-level discussion of classifier performance.

The final 6-gene panel includes several genes associated with interferon-related immune activity, consistent with established SLE blood-transcriptomic literature [7,8]. However, biological interpretation and external validation should be addressed separately in the thesis discussion and future validation work.

The performance metrics reported here are based on a sealed held-out donor cohort from the available dataset. Additional external validation would be required before claiming clinical diagnostic generalizability, in line with clinical prediction-model reporting recommendations [1].
