# SLE ML Pipeline — Output Summary

This folder contains the main output files generated by the donor-stratified machine-learning pipeline developed to identify and evaluate a compact diagnostic gene signature for systemic lupus erythematosus (SLE).

## Pipeline Overview

The original NCBI dataset contains logitudinal microarray data, meaning repeated samples from the same donors across multiple timepoints. If samples from the same donor were split across training and testing subsets, the resulting performance estimates could be optimistically biased due to donor-level information leakage [1,2].

To reduce this risk, donor identity was preserved during splitting and validation. 80& of donors were kept to develop the ML pipeline, while 20% were held out until the end to test it. The held-out donor set was sealed after the initial split and was not used for feature selection, classifier tuning, benchmarking, or final panel construction. This approach is aligned with general recommendations for avoiding selection bias and overfitting in prediction-model development, especially in high-dimensional gene-expression settings [1].

The main pipeline stages are:

```text
Input DEG + metadata matrix
        ↓
Donor-aware 80/20 development/held-out split
        ↓
Development-set feature-selector/classifier benchmarking
        ↓
Final compact gene-panel derivation
        ↓
Sealed held-out evaluation
        ↓
Uncertainty analysis and SHAP-based interpretation
```

## Section 2 — Development-Set Algorithm Benchmarking

Only the 80% of donors, refered to as the development cohort, was used to develop the ML pipeline, and specifically during algorithm benchmarking. The aim of this stage was to compare different feature-selector/classifier combinations. 

The development benchmark used an outer donor-aware 5-fold STRATIFIED cross-validation scheme, by applying scikit-learn's "StratifiedGroupKFold" [3-5]. Within each outer training split, a bootstrap Random Forest stability filter was applied before feature selection. This step prioritized genes that were repeatedly selected across donor-level resampling iterations, following the general rationale of stability-based selection strategies [6,7].

Within each outer fold:

1. The outer training donors were used for bootstrap RF stability filtering.
2. Each feature-selection method was applied to the stabilized training data.
3. Each classifier was tuned using inner donor-aware cross-validation.
4. The tuned model was refitted on the outer training donors.
5. The model was evaluated on the outer testing donors.
6. Outer-fold ROC AUC values were collected.

This nested structure reduces the risk of overfitting the model‑selection criterion and the associated selection bias in performance estimates that arises when tuning or feature selection is performed outside the validation loop [8].

A grid-based benchmarking design was used to systematically evaluate combinations of feature-selection methods (mRMR, LASSO, Elastic Net, SVM-RFE, Random Forest importance, Boruta) and classifier families (support vector machines, logistic regression, Random Forest, XGBoost, Naive Bayes, k-nearest neighbours). This approach allows direct comparison of linear and non-linear models coupled with sparse and non-sparse selectors under the same donor-aware cross-validation scheme, similar to recent benchmark studies that survey multiple feature-selection and classification pipelines in high-dimensional omics data [9,10]. By exploring the full selector–classifier grid rather than relying on a single default algorithm, the pipeline identifies combinations that are genuinely well-matched to the structure of the SLE transcriptomic data and reduces the risk of over-interpreting the performance of any single model choice [11].

## Section 3 — Final Gene-Panel Derivation

After selecting the best-performing feature-selector/classifier combination, the winning selector was rerun to train on the full development cohort to obtain a global ranking of candidate genes (because of the way CV works, each time it was previously run was only in a subset of the development dataset).

The final gene-panel derivation used:

- Boruta-based feature ranking;
- full-development bootstrap RF stability analysis;
- high-frequency gene retention;
- Pearson correlation-based redundancy pruning.

This stage was designed to derive a compact SLE gene signature that remains predictive while reducing redundancy among highly correlated genes, in line with previous work using gene-expression signatures to predict disease activity [14]. The goal of the gene signature panel was to preserved predictive signal while reducing redundancy among highly correlated genes, so correlation-based pruning was used to avoid retaining multiple similar predictors [13, 15].

### Bootstrap Stability

The full-development bootstrap RF stability analysis estimated per-gene selection frequencies, treating stability as the robustness of the selected gene set to perturbations of the training data, in line with standard definitions of feature-selection stability in high-dimensional settings. Genes with selection frequency greater than or equal to 0.80 were retained as high-frequency candidates [16].

### Permutation Test

A donor-stratified permutation test was used to estimate a null AUC distribution and compare the observed development-set AUC against chance-level label assignments. Permutation testing provides a model-performance significance check by comparing the observed score to the distribution of scores obtained under repeated randomization of the outcome labels [22,23].

Output files:

- [`04_permutation_meta.json`](<04_permutation_meta.json>)
- [`04_permutation_null.csv`](<04_permutation_null.csv>)

Permutation-test summary:

| Metric | Value |
|---|---:|
| Observed AUC | 0.900 |
| Null mean AUC | 0.504 |
| Number of permutations | 200 |
| Permutation p-value | 0.00498 |

![Permutation-test null distribution](<figures/04_permutation_histogram.png>)

The observed development-set AUC of 0.900 (vertical dashed line) lies in the extreme upper tail of the null AUC distribution (null mean 0.504), corresponding to a permutation p-value of 0.00498, which indicates that such performance would be unlikely under random label assignments [22,23].

The histogram shows the null AUC distribution from permuted labels, with the observed AUC marked as a vertical dashed line.

## SHAP-Based Interpretation

SHAP-based interpretation was used to summarize the contribution of each final-panel gene to the model output. SHAP (SHapley Additive exPlanations) assigns each feature a local contribution value for individual predictions and can be aggregated to obtain global summaries of feature importance, making it a commonly used approach for interpreting complex machine-learning models [24].

Output files:

- [`04_shap_summary.csv`](<04_shap_summary.csv>)
- [`04_shap_values.csv`](<04_shap_values.csv>)

![Top SHAP feature importances](<figures/04_shap_barplot.png>)

The SHAP barplot ranks final-panel genes by mean absolute SHAP value, providing a global view of which genes have the largest average impact on the model prediction. In this run, `IFI44L` showed the highest mean absolute SHAP contribution among the retained panel genes [24].

### SHAP Beeswarm Plot

![SHAP beeswarm plot](<figures/04_shap_beeswarm.png>)

The SHAP beeswarm plot shows, for each final-panel gene, the distribution of sample-level SHAP values along the x-axis (magnitude and direction of its effect on the prediction), with points vertically aligned to indicate density and colored by the underlying gene-expression value, so that high vs low expression can be visually linked to positive or negative shifts in the model output.

## Methodological Safeguards

Overall, several safeguards were used to reduce overfitting and biased performance estimation:

1. **Donor-aware splitting**  
   Samples from the same donor were kept together during development/held-out splitting and cross-validation.

2. **Sealed held-out evaluation**  
   The held-out cohort was not used during feature selection, model tuning, or final panel construction.

3. **Nested cross-validation during benchmarking**  
   Inner cross-validation was used for hyperparameter tuning, while outer cross-validation was used for performance estimation.

4. **Bootstrap RF stability filtering**  
   Repeated bootstrap resampling was used to prioritize stable candidate genes.

5. **Correlation-based redundancy pruning**  
   Pearson correlation filtering reduced redundancy in the final panel.

6. **Permutation testing**  
   A null AUC distribution was generated to assess whether observed model performance exceeded chance-level expectations.

7. **Transparent comparator model**  
   A logistic-regression comparator was trained and evaluated on the same final gene panel.

8. **SHAP interpretation**  
   SHAP analysis was used to summarize feature contributions for the final model.
