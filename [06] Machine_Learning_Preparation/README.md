# Pre-ML Activities

## Notes

This directory contains the **data preparation steps required before the Machine Learning (ML) analysis** of the transcriptomic dataset.

The matrices used here originate from the **LIMMA differential expression workflow performed in Step [08] of the repository pipeline**. That step produced the **set of Differentially Expressed Genes (DEGs)** that are used as **candidate features for the ML models**.

This stage focuses on constructing a **machine-learning-ready dataset** by:

- loading the DEG expression matrix
- integrating sample metadata
- ensuring correct sample annotations
- preparing the final table used by downstream ML scripts

Because the dataset is **longitudinal**, with multiple samples collected from the **same donor at different pregnancy timepoints**, the metadata integration is essential for preventing **data leakage during ML training**.

# Folder Contents

This directory contains the files required to construct the **ML-ready dataset**.

### Script

**ML_matrix_creation.ipynb**

Notebook that builds the machine-learning input matrix by:

- loading DEG expression data
- loading sample metadata
- merging the datasets
- organizing the final table structure

This script produces the **final dataset used by the ML pipeline**.

---

### Input Files

**ML_FeatureMatrix_AllDEGs_log2.tsv**

Expression matrix containing **log₂-transformed expression values** for all genes selected as **Differentially Expressed Genes (DEGs)** in the LIMMA analysis.

Structure:

- Rows → samples (GSM IDs)
- Columns → DEG features
- Values → log₂ expression values

This file serves as the **feature matrix for the ML models**.

---

**GSE108497_updated_metadata_II.csv**

Sample metadata for the GEO dataset.

Key annotations include:

- Sample identifier (GSM ID)
- Donor identifier
- Disease status (Healthy / SLE)
- Pregnancy timepoint

This metadata is required to properly label samples and track repeated measurements from the same individual.

---

### Output File

**ML_FeatureMatrix_AllDEGs_with_Metadata.tsv**

Final dataset produced by the preprocessing script.

This table combines:

- metadata columns
- DEG expression features


This format allows ML models to access both **sample labels** and **gene expression features** while preserving donor information for proper validation strategies.

# Overview of the Pre-ML Workflow

The preprocessing script performs the following steps.

---

## 1. Load DEG Expression Matrix

The DEG feature matrix generated by the LIMMA workflow is loaded into memory.

At this stage:

- rows represent **samples**
- columns represent **DEG features**

The sample identifier is stored as `SampleID` to enable merging with metadata.

---

## 2. Load Metadata

The sample metadata file is loaded and relevant columns are selected.

These columns describe:

- donor identity
- disease status
- pregnancy timepoint

This information provides the **labels required for ML classification tasks**.

---

## 3. Metadata Standardization

Metadata columns are renamed for clarity and consistency.

Example renaming:

| Original column | New column |
|---|---|
| Sample | SampleID |
| Donor_id | DonorID |
| sle | Condition |
| time_point | Timepoint |

Additional readable labels are also generated for clarity, such as:

- `Condition_label` (Healthy / SLE)
- `Time_label` (formatted timepoint)

---

## 4. Merge Expression Data with Metadata

The DEG expression matrix and metadata are merged using the **SampleID** column.

Only samples present in both datasets are retained.

This produces a unified dataset that links **gene expression measurements with biological annotations**.

---

## 5. Column Organization

Metadata columns are placed at the beginning of the table, followed by gene expression features.

This structure improves readability and facilitates downstream ML workflows.

---

## 6. Dataset Validation

Basic checks are performed to ensure dataset integrity.

These checks include verifying:

- number of samples
- number of gene features
- distribution of disease labels
- number of unique donors

Such validation steps help ensure the dataset is correctly constructed before proceeding to ML analysis.

---

# Role in the Repository

This directory acts as the **bridge between statistical transcriptomic analysis and machine learning modeling**.

The output dataset created here is used directly as input for the **Machine Learning pipeline**, where models are trained to distinguish between **SLE and healthy pregnancy samples** based on transcriptomic signatures.

---

# Important Considerations

Because the dataset contains **repeated measurements from the same donor**, ML pipelines must ensure that:

- samples from the **same donor are not split across training and testing sets**
- validation strategies respect **subject-level independence**

This prevents **information leakage** and ensures reliable model evaluation.
